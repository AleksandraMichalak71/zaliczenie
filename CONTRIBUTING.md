# CONTRIBUTING

## Zasady współpracy

1. Utwórz własny fork repozytorium.
2. Sklonuj fork na swój komputer.
3. Utwórz nowy branch przeznaczony do realizacji konkretnej zmiany.
4. Wprowadzaj zmiany w oddzielnych commitach z czytelnymi komunikatami.
5. Przed wysłaniem zmian upewnij się, że projekt działa poprawnie.
6. Utwórz Pull Request zawierający opis wykonanych zmian.


## Commity

Komunikaty commitów powinny być krótkie i opisywać zakres zmian.

Przykłady:

```text
Dodano sekcję o lwie
Dodano zdjęcie lwa
Zaktualizowano README
Utworzono CONTRIBUTING.md
```
